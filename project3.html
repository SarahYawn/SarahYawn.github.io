Project 2: Sentiment Analysis

Sentiment Analysis using NLTK and SpaCy

Natural Language Processing & Text Analysis

Author: Sarah Yawn

Date: July 17, 2025

Assignment: 7.1

Data Source: Kaggle - Emotions Dataset

This project explores sentiment analysis and natural language processing using two powerful Python libraries: NLTK (Natural Language Toolkit) and SpaCy. The analysis is performed on an emotions dataset containing text labeled with six sentiment categories: sadness (0), joy (1), love (2), anger (3), fear (4), and surprise (5). The project demonstrates both sentiment intensity analysis using VADER and advanced linguistic analysis including Named Entity Recognition (NER), noun phrase extraction, token-level analysis, and dependency parsing visualization.

Technologies & Libraries:

Python NLTK SpaCy Pandas KaggleHub Google Colab VADER Sentiment Analyzer en_core_web_sm

Technical Concepts Demonstrated:

  • Sentiment Analysis with NLTK: Implementation of VADER (Valence Aware Dictionary and sEntiment Reasoner) sentiment intensity analyzer to calculate polarity scores including negative, neutral, positive, and compound sentiment scores; interpretation of compound scores to classify overall sentiment as Positive (≥0.05), Negative (≤-0.05), or Neutral
  • Data Acquisition: Automated dataset download using KaggleHub API; dynamic CSV file discovery through directory traversal; random text selection for analysis demonstrating reproducible research practices
  • Named Entity Recognition (NER): SpaCy's entity extraction to identify and classify named entities such as persons, organizations, locations, dates, and numerical values in text; analysis of entity types and frequencies
  • Noun Phrase Extraction: Identification and extraction of noun chunks using SpaCy's syntactic parser; analysis of subject matter and themes through noun phrase frequency
  • Token-Level Linguistic Analysis: Detailed examination of individual tokens including lemmatization (base form), Part-of-Speech (POS) tagging, fine-grained tag classification, dependency relations, and word shape analysis
  • Dependency Parsing: Visualization of grammatical relationships between words using SpaCy's displacy module; creation of interactive dependency parse trees showing subject-verb-object relationships and modifiers
  • Theme Identification: Custom algorithm combining named entities and noun phrases with frequency counting to identify prevalent themes and topics in text; ranking of keywords by occurrence

Analytical Methods:

  • VADER Polarity Scoring: Calculated sentiment scores showing negative (0.268), neutral (0.732), positive (0.0), and compound (-0.4449) values for sample text "i couldnt feel any divine being in my own pulse"; comparison with dataset label (joy=1) revealing potential mislabeling in source data
  • Comparative Analysis: Human interpretation aligned with VADER's negative sentiment classification rather than dataset's joy label, suggesting NLTK's robust sentiment detection capabilities for complex emotional expressions
  • Entity Analysis: Extracted entities with type classification (e.g., "one" as CARDINAL number); empty entity results for certain texts indicating subjective, introspective content without external references
  • Noun Phrase Frequency: Identified recurring pronouns ("i" appearing twice) and key concepts ("a post", "marriage") suggesting personal narrative and self-referential themes
  • Syntactic Relationship Mapping: Dependency parse revealed complex sentence structures with multiple clauses, auxiliary verbs, and complementation patterns including ROOT verb "avoided", subject "i", and complement clauses with "because" conjunction
  • Theme Extraction Results: Top themes from sample text "i have avoided writing a post about marriage because i feel inadequate to write one
