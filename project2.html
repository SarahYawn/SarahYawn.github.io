Project 2: Wisconsin Weather Precipitation Prediction

Machine Learning & Interactive Dashboard

Author: Sarah Yawn

Date: July 20, 2025

Data Source: Kaggle - Wisconsin Weather Dataset

Comprehensive machine learning application predicting hourly precipitation using weather data from three Wisconsin stations (Southern Wisconsin Regional Airport, Green Bay Austin Straubel International Airport, and Appleton International Airport). This project implements a complete ML pipeline including extensive data cleaning of 854,646 observations, feature engineering, model selection and tuning, and deployment through an interactive Dash application with real-time prediction capabilities and historical trend visualization.

Technologies & Libraries:

Python Pandas NumPy Scikit-learn Matplotlib Seaborn Plotly Dash Dash Bootstrap Components SciPy Google Colab

Technical Concepts Demonstrated:

  • Data Acquisition & Management: Google Drive integration for dataset access, handling multiple CSV files (2P2.csv, GRB.csv, ATW.csv) with automated concatenation and source tracking
  • Data Cleaning & Preprocessing: Comprehensive handling of 27 columns with missing values using median imputation for numerical features and mode for categorical features; datetime conversion for temporal analysis; removal of empty columns and handling mixed data types
  • Feature Engineering: Created Temperature_Humidity_Category feature combining temperature and dew point differentials; extracted temporal features (year, month, day, hour, dayofweek); implemented binning for top 5 correlated features using quantile-based categorization; applied Yeo-Johnson power transformation to 11 skewed numerical features
  • Exploratory Data Analysis: Statistical profiling across 30 features; distribution analysis via histograms and count plots; correlation matrix analysis revealing high correlations (tmpf-dwpf: 0.92, tmpf-feel: 0.98); identification of multicollinearity issues
  • Outlier Detection & Removal: Z-score analysis identifying outliers in wind speed, gust, and ice accretion variables; systematic removal of 27,064 rows (3.17%) exceeding Z-score threshold of 3; boxplot visualization for outlier identification
  • Feature Selection: Removed highly correlated redundant features (feel, mslp) based on correlation threshold >0.9; dropped high-cardinality categorical features (metar, peak_wind_time, wxcodes); one-hot encoding for remaining categorical variables resulting in 342 engineered features
  • Model Selection & Evaluation: Comparative analysis of Linear Regression, Gradient Boosting Regressor, and Random Forest Regressor using 3-fold cross-validation; Random Forest selected as best model with lowest CV MSE (0.6375)
  • Hyperparameter Tuning: RandomizedSearchCV implementation with 10 iterations testing n_estimators (50-150), max_depth (10, 20, None), min_samples_split (2, 5), min_samples_leaf (1, 2), and bootstrap options; optimal parameters: n_estimators=100, min_samples_split=5, min_samples_leaf=2, max_depth=None, bootstrap=True
  • Model Performance Assessment: Multiple evaluation metrics including MSE (0.598), R² (0.382), residual analysis, and feature importance ranking; visualization through actual vs. predicted scatter plots and residual distribution analysis
  • Interactive Dashboard Development: Dash application with Bootstrap components featuring real-time precipitation prediction interface, historical trend visualization, station-specific current conditions display, and dynamic feature input forms
  • Data Visualization: Comprehensive visualizations including correlation heatmaps, feature importance bar charts, residual plots, distribution histograms, and time-series line graphs using Matplotlib, Seaborn, and Plotly

Analytical Methods:

  • Correlation Analysis: Pearson correlation matrix revealing strongest predictors of precipitation: relative humidity (0.227), dew point (0.059), wind speed (0.056); identification of multicollinearity issues requiring feature removal
  • Statistical Testing: Z-score based outlier detection with threshold of 3 standard deviations; analysis of feature distributions to inform transformation strategies
  • Time Series Feature Engineering: Extraction of cyclical temporal patterns through year, month, day, hour, and day-of-week features; consideration of seasonal and daily patterns in weather data
  • Ensemble Learning: Random Forest implementation leveraging 100 decision trees with bootstrap aggregating; comparison with gradient boosting and linear models
  • Cross-Validation: 3-fold cross-validation for robust model evaluation; 80/20 train-test split (662,065 training samples, 165,517 test samples) for final performance assessment
  • Feature Importance Analysis: Identification of top predictive features: visibility (15.9%), relative humidity (13.1%), year (9.7%), hour (7.1%), wind direction (7.0%); insights informing model interpretation
